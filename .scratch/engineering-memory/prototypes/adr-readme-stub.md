# Architecture Decision Records

Hard decisions that override soft Architecture Bias live here as ADRs.
Existing ADRs are never rewritten by Memory Install. Do not pre-seed sample ADRs.

Name files `NNNN-kebab-title.md` (e.g. `0001-event-sourced-orders.md`).
Keep each ADR thin: short title + 1–3 sentences (context, decision, why). Optional Status / Considered Options / Consequences only when they add real value.
Link from prose when an agent must not re-litigate the decision. Offer ADRs via `/domain-modeling` when a hard trade-off crystallises.
